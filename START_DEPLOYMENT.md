## ðŸ“‹ **STEP-BY-STEP (Manual)**

If you prefer manual deployment, follow these steps:

### **STEP 1: Install CLIs** (2 minutes - one time only)

```powershell
# Install Railway CLI
npm install -g @railway/cli

# Install Vercel CLI
npm install -g vercel

# Login to both
railway login
vercel login
```

### **STEP 2: Create Railway Project** (3 minutes - one time only)

1. Go to https://railway.app
2. Click "New Project"
3. Select "Empty Project"
4. Name it "realco-backend"
5. Click "Add PostgreSQL" (in project dashboard)

### **STEP 3: Set Environment Variables** (5 minutes - one time only)

**Option A: Automated Script**
```powershell
cd backend\scripts
.\setup-railway-env.ps1
```

**Option B: Manual**
```powershell
# Generate secrets
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
# Copy output as JWT_SECRET

# Set in Railway
railway variables set JWT_SECRET="YOUR_GENERATED_SECRET"
railway variables set ENCRYPTION_KEY="YOUR_GENERATED_SECRET"

# Add your API keys
railway variables set STRIPE_SECRET_KEY="sk_live_..."
railway variables set PLAID_CLIENT_ID="..."
railway variables set PLAID_SECRET="..."
railway variables set AWS_ACCESS_KEY_ID="..."
railway variables set AWS_SECRET_ACCESS_KEY="..."
```

**See complete list:** `RAILWAY_SETUP_GUIDE.md`

### **STEP 4: Deploy Backend** (2 minutes)

```powershell
cd backend

# Generate Prisma client
npx prisma generate

# Run migrations
railway run npx prisma migrate deploy

# Deploy
railway up
```

### **STEP 5: Deploy Frontend** (2 minutes)

```powershell
cd frontend

# Link project (first time only)
vercel link

# Set environment variable
vercel env add VITE_API_URL production
# Enter your Railway URL: https://your-app.railway.app/api/v1

# Deploy
vercel --prod
```

### **STEP 6: Verify** (1 minute)

```powershell
# Check backend health
railway logs

# Check frontend
vercel logs

# Test in browser
start https://your-app.vercel.app
```
